List of algorithms

All algorithms are accessible as static methods of class com.tknkla.rj.RJ. Majority of the algorithms are listed and explained here.

Parallel execution

Majority of the algorithms are transparently parallelized. Parallelization is disabled by default but can be enabled selecting an approriate ExecutionStrategy.

Configuration of parallel execution:

  • setExecutor(ExecutionStrategy)
  • getExecutor()

Note that the parallelization setting is global. The recommended way to configure parallelization is to select the execution strategy during application startup (or in the case of a programming library it is not recommended to configure it at all but leave it to the application using the library). Reconfiguration of the execution strategy is strongly discouraged unless it can be ensured that no other thread is using any of the parallel algorithms in RJ at the same time. More dedicate control over parallelization can be achieved with a custom implementation of ExecutionStrategy.

Provided execution strategies:

  • ExecutionStrategy.LOCAL (no parallelization; the default option)
  • ExecutionStrategy.PARALLEL (default parallelization using the shared ForkJoinPool)

Generic parallel execution:

  • execute(int, int, IntConsumer)
  • execute(int, int, int, IntUnaryOperator, IntBinaryOperator)
  • execute(int, int, long, IntToLongFunction, LongBinaryOperator)
  • execute(int, int, Object, IntFunction, BinaryOperator)

Sorting and grouping

Sorting and grouping algorithms which generate a preorder (array of arrays of symbols) are named 'groups' while algorithms, which generate an order (array of symbols), are named 'order'.

Computation of absolute preorder:

  • groups(int, IntToLongFunction, LongToIntBinaryOperator, LongToIntBinaryOperator)
  • groups(int, IntUnaryOperator, IntBinaryOperator, IntBinaryOperator)
  • groups(Class, int, IntFunction, Comparator, Comparator)

Computation of absolute preorder within a preorder:

  • groups(int[][], IntBinaryOperator, IntBinaryOperator)
  • groups(long[][], LongToIntBinaryOperator, LongToIntBinaryOperator)
  • groups(Class, Object[][], Comparator, Comparator)

Computation of absolute order:

  • order(int, IntToLongFunction, LongToIntBinaryOperator, LongBinaryOperator)
  • order(int, IntUnaryOperator, IntBinaryOperator, IntBinaryOperator)
  • order(Class, int, IntFunction, Comparator, BinaryOperator)

Computation of truly relative order:

  • groups(int[][], IntBinaryOperator, IntGroupOperator, IntBinaryOperator)
  • groups(long[][], LongBinaryOperator, LongGroupOperator, LongToIntBinaryOperator)
  • groups(Class, Object[][], BiFunction, GroupOperator, Comparator)

Computation of strongly relative order:

  • order(int[][], IntBinaryOperator, IntGroupOperator, IntBinaryOperator)
  • order(long[][], LongBinaryOperator, LongGroupOperator, LongToIntBinaryOperator)
  • order(Class, Object[][], BiFunction, GroupOperator, Comparator)

Order propagation (the relative part of computation of relative order):

  • propagate(int[][], IntBinaryOperator, IntBinaryOperator, IntGroupOperator, Predicate)
  • propagate(long[][], LongBinaryOperator, LongToIntBinaryOperator, LongGroupOperator, Predicate)
  • propagate(Class, Object[][], BiFunction, Comparator, GroupOperator, Predicate)

Comparision

Relative comparision between vertices:

  • compare(int, int, int[][], IntBinaryOperator, IntGroupOperator)
  • compare(long, long, long[][], LongBinaryOperator, LongGroupOperator)
  • compare(Class, Object, Object, Object[][], BiFunction, GroupOperator)

Absolute comparision of arrays:

  • compare(int[], int[], IntBinaryOperator)
  • compare(long[], long[], LongToIntBinaryOperator)
  • compare(Object[], Object[], Comparator)

Combining comparisions:

  • compare(int, IntUnaryOperator)
  • compare(IntSupplier...)

Elementary array operations

Anary set operations for ordered arrays:

  • merge(int, IntFunction, IntBinaryOperator, IntBinaryOperator, SetOperator)
  • merge(int, IntFunction, LongToIntBinaryOperator, LongBinaryOperator, SetOperator)
  • merge(Class, int, IntFunction, Comparator, BinaryOperator, SetOperator)

Binary set operations for ordered arrays:

  • merge(int[], int[], IntBinaryOperator, IntBinaryOperator, SetOperator)
  • merge(long[], long[], LongToIntBinaryOperator, LongBinaryOperator, SetOperator)
  • merge(Class, Object[], Object[], Comparator, BinaryOperator, SetOperator)

Array creation and population (1d):

  • populate(int, IntToLongFunction)
  • populate(int, IntUnaryOperator)
  • populate(Class, int, IntFunction)

Array creation and population (2d):

  • populate(int, int, IntBinaryOperator)
  • populate(int, int, IntToLongBinaryOperator)
  • populate(Class, int, int, IntBiFunction)

Anary array concatenation:

  • joinAsInt(int, IntFunction)
  • joinAsLong(int, IntFunction)
  • join(Class, int, IntFunction)

Binary array concatenation:

  • join(int[], int[])
  • join(long[], long[])
  • join(Object[], Object[])

Trinary array concatenation:

  • join(int[], int[], int[])
  • join(long[], long[], long[])
  • join(Object[], Object[], Object[])

Searching

Abstract binary search:

  • search(int, int, IntUnaryOperator)

Abstract linear search (first match and best match):

  • search(int, int, IntPredicate)
  • search(int, int, IntBinaryPredicate)

Graph utilities

Decoding of undirected graph (from binary):

  • decodeUndirected(BigInteger)
  • decodeUndirected(IntPredicate)
  • verticesOfUndirected(BigInteger)
  • verticesOfUndirected(int)

Encoding of undirected graph (to binary):

  • encodeUndirected(int, IntBinaryPredicate)
  • encodeUndirected(int, IntBinaryPredicate, IntConsumer)

Lambda wrappers for conversions between undirected and abstract graphs:

  • asOperator(IntBinaryPredicate)
  • asOperator(IntBinaryPredicate, int, int, int)
  • asPredicate(IntBinaryOperator, IntPredicate)

Randomization

Random binary generation:

  • randomBinary(int, double, double, Random)

Random order generation:

  • randomOrder(int, Random)

Random (undirected) graph generation (symmetric and asymmetric):

  • randomUndirected(int, double, double, Random)
  • randomUndirectedAsymmetric(int, double, Random)
  • randomUndirectedSymmetric(int, double, Random)

Shuffling:

  • shuffle(int[], Random)
  • shuffle(long[], Random)
  • shuffle(Object[], Random)
  • shuffle(int, int, IntBinaryConsumer, Random)
